Project scope

Expected: discovery → pilot → production rollout

Core use cases

  • Find duplicates / near-duplicates across existing designs to reduce rework and improve reuse.
  • Search by image or sketch (uploaded or pasted) to retrieve visually similar designs.
  • Search by 3D CAD (CAD file or derived representation) to retrieve similar geometry.
  • Auto-tagging to populate attributes and standardize the design catalog.
  • Catalog standardization with governance (taxonomy, confidence, review workflow).

Out of scope (unless proposed)

  • End-to-end eCommerce storefront build.
  • Replacing existing PLM/ERP/CAD systems.
  • Manual re-labeling of the full library without AI-assisted workflows.

Functional requirements

Tell us what you meet out-of-the-box vs custom

Visual similarity search (2D)

  • Query by photo and sketch with top‑K results and similarity scores.
  • Filters/facets (metal, stone type, collection, style, category, etc.) based on tags/metadata.
  • Configurable thresholding for “same design”, “variant”, and “similar inspiration”.
  • Batch mode to discover duplicates and clusters across large sets.

3D similarity search (CAD)

  • Ingest 3D CAD assets (or derived meshes/renders) to enable similarity by geometry.
  • Support for normalization (scale/orientation) and robust matching for minor edits.
  • Explainability or diagnostics (e.g., similar substructures / global shape match indicators) preferred.

Auto-tagging & catalog standardization

  • Auto-suggest tags with confidence scores and human review workflow.
  • Taxonomy management (controlled vocabulary, synonyms, hierarchy).
  • Standardized outputs for a digital catalog (attributes, naming conventions, categories).
  • Audit trail: who approved what, when, and model version.

Quality, scale & operations

  • Handles ~300k designs with room to grow (index build times, incremental updates, re-index strategy).
  • Clear performance targets (latency, throughput) and monitoring (SLAs/SLOs).
  • Role-based access, permissions, and environment separation (dev/stage/prod).
  • Data governance: retention, access logging, model retraining approach.

Integration requirements

CAD/PLM/ERP connectivity + import/export

Systems & interfaces

  • Integration with existing CAD tools and repositories (ingest + links back to source).
  • Integration with PLM / ERP for item master / attributes / approvals.
  • APIs: REST (required), webhooks/eventing (preferred).
  • Bulk import/export: CSV/JSON + object storage for assets.

Security & deployment

  • SSO (SAML/OIDC) and RBAC.
  • Encryption in transit and at rest; key management approach.
  • Deployment options: vendor-hosted SaaS or customer VPC/on‑prem (please specify).
  • Compliance posture (e.g., SOC2/ISO) if applicable.

Expected deliverables

Propose phased delivery with measurable outcomes

Pilot (recommended)

  • Discovery + data sampling plan + success metrics (precision/recall, dedupe hit rate, reviewer time saved).
  • Prototype UI or workflow integrated into existing process.
  • Benchmark report vs baseline (manual search / current tags).
  • Production architecture proposal + rollout plan.

Production rollout

  • Search services (2D + 3D), indexing pipeline, monitoring, and admin tools.
  • Auto-tagging pipeline with human-in-the-loop review.
  • Integrations (CAD/PLM/ERP), data contracts, and documentation.
  • Enablement: training, runbooks, SLAs, support model.
